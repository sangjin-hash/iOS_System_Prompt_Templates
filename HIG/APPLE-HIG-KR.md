## 디자인 원칙

### 1. Clarity (명확성)
- **텍스트의 가독성**: 모든 크기의 텍스트가 읽기 쉬워야 함
- **아이콘의 명확성**: 아이콘은 정확하고 이해하기 쉬워야 함
- **장식의 최소화**: 기능이 콘텐츠를 방해하지 않아야 함
- **중요한 기능 강조**: 사용자가 필요한 기능을 쉽게 찾을 수 있어야 함

### 2. Deference (존중)
- **콘텐츠 우선**: UI는 콘텐츠를 돋보이게 하고 방해하지 않아야 함
- **반투명 효과 활용**: 배경을 흐리게 하여 콘텐츠를 강조
- **여백 활용**: 적절한 여백으로 중요한 요소 강조
- **제스처 존중**: 사용자의 제스처에 즉각적으로 반응

### 3. Depth (깊이)
- **시각적 계층**: 레이어와 모션을 통해 계층 구조 표현
- **애니메이션**: 화면 전환 시 부드러운 애니메이션 제공
- **그림자와 흐림**: 깊이감을 표현하여 UI 계층 구조 명확화

## iOS 플랫폼 특성

### 화면 크기와 해상도
- **다양한 디바이스 지원**: iPhone (다양한 크기), iPad, iPhone Pro Max 등
- **Safe Area**: 노치, Dynamic Island, 홈 인디케이터를 고려한 안전 영역 준수
- **Auto Layout**: 다양한 화면 크기에 대응하는 적응형 레이아웃
- **Dynamic Type**: 사용자 설정에 따라 텍스트 크기 자동 조절

### 인터페이스 요소
- **Status Bar**: 시간, 배터리, 신호 강도 등을 표시하는 상단 영역
- **Navigation Bar**: 화면 상단의 내비게이션 요소
- **Tab Bar**: 화면 하단의 주요 섹션 전환 메뉴
- **Toolbar**: 현재 화면의 액션을 수행하는 도구 모음

### 제스처
- **Tap**: 버튼, 컨트롤, 항목 선택
- **Drag**: 스크롤, 패닝
- **Swipe**: 리스트 항목 액션, 화면 전환
- **Pinch**: 확대/축소
- **Long Press**: 컨텍스트 메뉴 표시
- **Pull to Refresh**: 콘텐츠 새로고침

## 앱 아키텍처

### 앱 실행 (Launch)
- **빠른 실행**: 앱은 가능한 빨리 실행되어야 함
- **Launch Screen**: 앱의 첫 화면과 유사한 정적 이미지 사용
  - 광고, 브랜딩, 스플래시 화면 금지
  - 텍스트나 로고는 최소화
- **초기 설정 최소화**: 필수 설정만 요구하고, 나머지는 나중에

### 온보딩 (Onboarding)
- **즉시 가치 제공**: 튜토리얼 없이도 사용 가능하도록
- **선택적 튜토리얼**: 복잡한 기능만 간단히 설명
- **인터랙티브 학습**: 정적 화면보다는 실제 사용 중 학습 유도
- **건너뛰기 옵션**: 사용자가 온보딩을 건너뛸 수 있어야 함

### 로딩
- **진행 상황 표시**: 긴 작업은 진행률을 표시
- **Activity Indicator**: 시간을 예측할 수 없는 작업
- **Progress Bar**: 시간을 예측할 수 있는 작업
- **콘텐츠 우선 로딩**: 중요한 콘텐츠부터 먼저 표시

### 업데이트 요청
- **강제 업데이트 최소화**: 중요한 보안 문제만 강제
- **업데이트 이점 설명**: 새로운 기능이나 개선 사항 안내
- **나중에 알림 옵션**: 사용자가 선택할 수 있도록

## 내비게이션

### 계층적 내비게이션 (Hierarchical Navigation)
- **설정 앱 스타일**: 상위 화면에서 하위 화면으로 드릴다운
- **Navigation Bar 사용**: Back 버튼으로 이전 화면 복귀
- **명확한 제목**: 각 화면의 제목을 명확히 표시
- **Large Title 활용**: 최상위 화면에서 큰 제목 사용

### 플랫 내비게이션 (Flat Navigation)
- **Tab Bar 사용**: 주요 섹션 간 빠른 전환
- **3~5개 탭 권장**: 너무 많은 탭은 혼란을 야기
- **현재 위치 표시**: 선택된 탭을 명확히 표시
- **일관된 탭 순서**: 앱 전체에서 탭 순서 유지

### 콘텐츠 주도 내비게이션 (Content-Driven Navigation)
- **게임이나 미디어 앱**: 콘텐츠 중심의 자유로운 이동
- **명확한 경로 제공**: 사용자가 현재 위치를 파악할 수 있도록
- **뒤로 가기 기능**: 이전 상태로 돌아갈 수 있는 방법 제공

### 모범 사례
- **예측 가능한 동작**: 표준 내비게이션 패턴 준수
- **명확한 Back 버튼**: 이전 화면 제목 표시
- **Page Control**: 동일 계층의 여러 화면 표시 (페이지네이션)
- **Segmented Control**: 밀접하게 관련된 콘텐츠 전환

## 모달리티 (Modality)

### 모달 사용 시기
- **중요한 정보 전달**: 사용자의 주의가 필요한 경우
- **복잡한 작업**: 여러 단계가 필요한 작업
- **명확한 선택 요구**: 사용자의 결정이 필요한 경우

### 모달 종류

#### 1. Alerts
- **중요한 메시지 전달**: 오류, 경고, 확인 요청
- **간결한 제목과 메시지**: 명확하고 이해하기 쉽게
- **1~2개의 버튼**: 너무 많은 옵션은 피하기
- **파괴적 액션 명확화**: 빨간색으로 강조

#### 2. Action Sheets
- **여러 선택지 제공**: 현재 작업과 관련된 옵션
- **취소 버튼 포함**: 사용자가 빠져나갈 수 있도록
- **iPad에서는 Popover 사용**: Action Sheet 대신

#### 3. Sheets (Modal Sheets)
- **전체 화면 작업**: 정보 입력, 설정 등
- **명확한 완료/취소 버튼**: 상단에 배치
- **변경 사항 확인**: 취소 시 확인 메시지 표시
- **Card 스타일**: iOS 13 이상에서 기본 스타일

#### 4. Popovers (iPad)
- **추가 정보나 옵션**: 컨텍스트 유지
- **외부 탭으로 닫기**: 사용자가 쉽게 닫을 수 있도록
- **화살표로 연결**: 어디서 나타났는지 명확히

### 모달 사용 원칙
- **남용 금지**: 필요할 때만 사용
- **명확한 닫기 방법**: 사용자가 어떻게 닫는지 알아야 함
- **작업 흐름 방해 최소화**: 가능하면 인라인으로 처리
- **스와이프로 닫기**: iOS 13 이상에서 아래로 스와이프 지원

## 인터랙션

### 터치와 제스처
- **44x44 포인트 터치 영역**: 최소 터치 가능 영역
- **표준 제스처 준수**: 사용자가 익숙한 제스처 사용
- **커스텀 제스처 교육**: 비표준 제스처는 사용법 안내
- **제스처 충돌 방지**: 시스템 제스처와 충돌하지 않도록

### 피드백
- **시각적 피드백**: 버튼 상태 변화, 하이라이트
- **촉각 피드백 (Haptics)**: Haptic Feedback으로 물리적 반응 제공
- **청각 피드백**: 중요한 액션에 사운드 추가 (선택적)
- **즉각적 반응**: 사용자 입력에 즉시 반응

### 애니메이션
- **목적 있는 애니메이션**: 시각적 연속성과 이해를 돕는 애니메이션
- **적절한 속도**: 너무 빠르거나 느리지 않게 (0.2~0.4초)
- **자연스러운 움직임**: ease-in-out 같은 자연스러운 곡선
- **감속 고려**: 설정에서 애니메이션 감소 옵션 존중

### 드래그 앤 드롭
- **명확한 드래그 대상**: 무엇을 드래그할 수 있는지 명확히
- **시각적 피드백**: 드래그 중 대상 하이라이트
- **드롭 가능 영역 표시**: 어디에 드롭할 수 있는지 표시
- **실패 처리**: 유효하지 않은 드롭은 원위치로 애니메이션

## 컴포넌트

### Buttons

#### System Buttons
- **명확한 레이블**: 동사로 시작하는 명확한 액션
- **적절한 크기**: 최소 44x44 포인트
- **상태 표시**: Normal, Highlighted, Disabled 상태 구분

#### Text Buttons
- **간단한 액션**: 주로 내비게이션이나 간단한 작업
- **SF Symbols 활용**: 아이콘과 텍스트 조합

#### Filled Buttons
- **주요 액션**: 화면에서 가장 중요한 액션
- **충분한 여백**: 주변 요소와 구분
- **하나의 주요 버튼**: 화면당 하나의 Filled Button 권장

#### Floating Action Button (FAB)
- **iOS에서는 비권장**: Android 패턴, iOS에서는 탭바나 툴바 사용

### Lists and Tables

#### List Styles
- **Plain**: 기본 리스트 스타일
- **Grouped**: 섹션으로 그룹화된 리스트
- **Inset Grouped**: 여백이 있는 그룹 리스트 (iOS 13+)
- **Sidebar**: iPad에서 사이드바용 리스트

#### 리스트 항목
- **명확한 계층**: 제목, 부제목, 상세 정보 구분
- **Disclosure Indicator**: 화살표로 더 많은 정보 표시
- **Accessory Views**: 체크마크, 정보 버튼 등
- **Swipe Actions**: 삭제, 편집 등의 액션 제공
- **Contextual Menus**: Long Press로 더 많은 옵션 제공

#### 성능
- **셀 재사용**: 스크롤 성능 최적화
- **이미지 비동기 로딩**: 네트워크 이미지는 백그라운드에서
- **적절한 셀 높이**: 자동 높이보다 고정 높이가 성능상 유리

### Text Fields and Text Views

#### Text Field
- **명확한 Placeholder**: 입력 예시 제공
- **Clear Button**: 입력 내용 빠르게 지우기
- **적절한 키보드 타입**: Email, Number, URL 등
- **Return Key 커스터마이징**: 다음, 완료, 검색 등
- **입력 검증**: 실시간 또는 제출 시 검증

#### Text View
- **여러 줄 입력**: 긴 텍스트 입력용
- **스크롤 가능**: 내용이 많을 경우 스크롤
- **자동 높이 조절**: 내용에 맞춰 높이 변경 (선택적)

#### 보안
- **비밀번호 필드**: Secure Text Entry 활성화
- **자동 완성**: 적절한 Content Type 설정
- **Two-Factor Authentication**: SMS 자동 입력 지원

### Search

#### Search Bar
- **명확한 위치**: 주로 Navigation Bar에 배치
- **Placeholder**: "검색" 또는 "OO 검색"
- **Cancel 버튼**: 검색 취소 옵션
- **Scope Bar**: 검색 범위 필터링 (선택적)

#### 검색 경험
- **즉시 결과 표시**: 입력하는 동안 실시간 검색
- **최근 검색**: 이전 검색어 표시
- **추천 검색**: 인기 검색어나 추천 검색어
- **결과 없음 처리**: 대안 제시

### Pickers

#### Date Picker
- **인라인 스타일**: iOS 14+ 기본 스타일
- **Wheels 스타일**: 드럼 롤 스타일 (레거시)
- **Compact 스타일**: 공간 절약형

#### Picker View
- **여러 컴포넌트**: 다중 선택 휠
- **명확한 선택**: 중앙 선택 영역 강조

### Switches and Toggles
- **On/Off 상태**: 설정 활성화/비활성화
- **즉각적 적용**: 토글 즉시 설정 변경
- **레이블 제공**: 무엇을 켜고 끄는지 명확히

### Sliders
- **연속적인 값**: 볼륨, 밝기 등
- **최소/최대 표시**: 범위를 명확히
- **실시간 피드백**: 슬라이더 이동 중 값 반영

### Steppers
- **이산적 값 조절**: +/- 버튼으로 값 증감
- **현재 값 표시**: 숫자로 현재 값 표시

### Progress Indicators

#### Progress Bar
- **확정적 진행**: 시간을 예측할 수 있는 작업
- **0~100% 표시**: 진행률 명확히

#### Activity Indicator
- **불확정적 진행**: 시간을 예측할 수 없는 작업
- **적절한 크기**: Small, Medium, Large

#### Progress Views
- **백그라운드 작업**: 다운로드, 업로드 등
- **취소 옵션**: 가능하면 취소 버튼 제공

### Segmented Controls
- **밀접한 관련 옵션**: 2~5개의 세그먼트
- **균등한 너비**: 가능하면 같은 너비로
- **명확한 레이블**: 짧고 명확한 텍스트

### Tab Bars
- **주요 섹션**: 앱의 최상위 내비게이션
- **3~5개 탭**: 너무 많으면 More 탭 사용
- **아이콘과 레이블**: 둘 다 제공 권장
- **현재 탭 표시**: 색상으로 구분

### Toolbars
- **현재 화면 액션**: 편집, 공유, 삭제 등
- **아이콘 버튼**: 텍스트보다 아이콘 선호
- **적절한 간격**: 버튼 사이 충분한 여백

### Navigation Bars
- **화면 제목**: 현재 화면 표시
- **Back 버튼**: 이전 화면으로 복귀
- **액션 버튼**: 주요 액션 (Done, Edit, Add 등)
- **Large Title**: 최상위 화면에서 사용

### Alerts and Action Sheets
- **간결한 제목**: 한 줄로 요약
- **설명 메시지**: 선택적, 필요시에만
- **명확한 버튼**: 액션이 명확한 레이블
- **파괴적 액션**: 빨간색으로 강조

### Sheets and Modals
- **명확한 목적**: 왜 모달이 나타났는지 명확히
- **완료/취소 버튼**: 상단에 배치

## 시각 디자인

### 색상 (Color)

#### 시스템 색상
- **Dynamic Colors**: Light/Dark Mode 자동 대응
- **Semantic Colors**: 의미에 따른 색상 (Label, Background, Fill 등)
- **Tint Color**: 앱의 주요 색상, 일관되게 사용

#### 색상 사용 원칙
- **명확한 계층**: 색상으로 중요도 구분
- **접근성 고려**: 충분한 대비율 (WCAG AA 이상)
- **색맹 고려**: 색상만으로 정보 전달하지 않기
- **브랜드 색상**: 앱의 정체성을 나타내는 색상

#### Light/Dark Mode
- **자동 전환**: 시스템 설정에 따라 자동 전환
- **Semantic Colors 사용**: 수동 변경 없이 자동 대응
- **Asset Catalog**: 이미지도 Light/Dark 버전 제공
- **테스트**: 두 모드 모두에서 테스트

### 타이포그래피 (Typography)

#### San Francisco 폰트
- **시스템 폰트**: SF Pro (iOS), SF Compact (watchOS)
- **Dynamic Type**: 사용자 설정에 따라 크기 자동 조절
- **Weight 활용**: Ultralight ~ Black (9단계)

#### Text Styles
- **Large Title**: 34pt, Bold
- **Title 1**: 28pt, Regular
- **Title 2**: 22pt, Regular
- **Title 3**: 20pt, Regular
- **Headline**: 17pt, Semibold
- **Body**: 17pt, Regular (기본 텍스트)
- **Callout**: 16pt, Regular
- **Subheadline**: 15pt, Regular
- **Footnote**: 13pt, Regular
- **Caption 1**: 12pt, Regular
- **Caption 2**: 11pt, Regular

#### 타이포그래피 원칙
- **가독성 우선**: 충분한 크기와 간격
- **일관성**: 같은 목적의 텍스트는 같은 스타일
- **계층 구조**: 크기와 무게로 중요도 표현
- **Dynamic Type 지원**: 사용자 설정 존중
- **줄 간격**: 적절한 line height
- **정렬**: 왼쪽 정렬 기본, 가운데 정렬은 제한적으로

### 아이콘과 이미지 (Icons and Images)

#### SF Symbols
- **시스템 아이콘**: 4,000개 이상의 벡터 아이콘
- **일관된 스타일**: San Francisco 폰트와 조화
- **크기 자동 조절**: Dynamic Type과 연동
- **Weight와 Scale**: 다양한 굵기와 크기 변형
- **Color**: 단색, 다색, 계층적 색상 지원
- **커스텀 Symbols**: 앱 전용 심볼 제작 가능

#### 앱 아이콘
- **1024x1024**: App Store용 고해상도
- **둥근 모서리 없음**: 시스템이 자동으로 적용
- **그래디언트와 그림자**: 적절히 사용
- **브랜드 정체성**: 앱을 대표하는 디자인
- **단순명료**: 복잡하지 않고 인식하기 쉽게
- **배경 채우기**: 투명 배경 금지

#### 이미지 사용
- **해상도**: @1x, @2x, @3x 제공
- **Asset Catalog**: 이미지 관리 도구 사용
- **압축**: 적절한 압축으로 용량 최적화
- **LazyLoading**: 필요할 때 로드
- **플레이스홀더**: 로딩 중 표시

#### 템플릿 이미지
- **Tint Color 적용**: 단색 아이콘
- **렌더링 모드**: Template vs Original

### 레이아웃 (Layout)

#### Safe Area
- **노치/Dynamic Island**: 상단 안전 영역
- **홈 인디케이터**: 하단 안전 영역
- **여백 고려**: Safe Area 내부에 중요 콘텐츠 배치

#### Auto Layout
- **적응형 레이아웃**: 다양한 화면 크기 대응
- **Constraint 기반**: 명확한 제약 조건
- **Size Classes**: Regular/Compact 조합
- **Dynamic Height**: 콘텐츠에 맞춰 높이 조절

#### 그리드와 정렬
- **일관된 간격**: 8pt 또는 4pt 배수 권장
- **여백**: 최소 16pt 좌우 여백
- **정렬**: 요소들을 정렬하여 깔끔한 디자인

#### 반응형 디자인
- **Portrait/Landscape**: 두 방향 모두 지원
- **iPad 멀티태스킹**: Split View, Slide Over 대응
- **Dynamic Type**: 텍스트 크기 변화 대응

## 접근성 (Accessibility)

### VoiceOver
- **레이블**: 모든 UI 요소에 명확한 레이블
- **힌트**: 필요시 추가 설명 제공
- **순서**: 논리적인 포커스 순서
- **그룹화**: 관련 요소를 그룹으로
- **커스텀 액션**: Swipe로 추가 액션 제공

### Dynamic Type
- **확장 가능**: 텍스트 크기 변화 대응
- **레이아웃 조정**: 큰 텍스트에도 깨지지 않게
- **스크롤 영역**: 잘린 텍스트를 스크롤로
- **최소 크기**: 중요 텍스트는 최소 크기 유지

### 색상과 대비
- **대비율**: 최소 4.5:1 (텍스트), 3:1 (UI 요소)
- **색맹 고려**: 색상만으로 정보 전달 금지
- **Dark Mode**: 충분한 대비 유지

### 터치 영역
- **최소 크기**: 44x44 포인트
- **충분한 간격**: 버튼 사이 여백
- **명확한 경계**: 터치 가능 영역 명확히

### 모션
- **Reduce Motion**: 애니메이션 감소 설정 존중
- **대체 효과**: 애니메이션 대신 페이드 등

### 접근성 검사
- **Accessibility Inspector**: Xcode 도구 활용
- **VoiceOver 테스트**: 실제로 VoiceOver로 테스트
- **Dynamic Type 테스트**: 다양한 크기로 테스트
- **색상 대비 확인**: 대비율 검사 도구 사용
